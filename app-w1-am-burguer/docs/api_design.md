# api_design.md
## ¿Para qué sirve este archivo?
Define los "puntos de conexión" entre el frontend y el backend. Es como un contrato entre las dos partes del sistema.

## ¿Qué va aquí?
- Lista de endpoints (URLs del sistema)
- Qué datos recibe cada uno y qué devuelve
- Ejemplo: POST /pedidos → recibe los productos del carrito, devuelve el ID del pedido
